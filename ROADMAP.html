CodeTrace_Git Roadmap

Git insight roadmap for the first public release.

The goal is to ship a clean .vsix-installable extension powered by the local Git CLI, delivered in 3 layers so complexity does not arrive too early: MVP to ship, MVP+ to improve usability, and Advanced for power-user capabilities.

Delivery Strategy Ship small, iterate fast
Core Backend Local Git CLI
Main UI TreeView + Decorations
Heavy UI Webview only when needed
Phase 1 Ship First

MVP

The first release should be strong enough for daily use in a normal Git repository.

Must Have

  • Inline Blame Low

    Show the last commit that changed the current line, with current-line or all-lines modes.

  • File Blame Medium

    Show full-file blame with line mapping for audit and debugging workflows.

  • Line Hover Low

    Show author, date, commit hash, and commit message when hovering a line.

  • CodeLens Medium

    Show recent commit insight above important functions or classes.

  • File History Low

    Provide a sidebar view for the active file history, capped by a configurable commit limit.

  • Commit Details Low

    Show author, message, changed files, patch, and stat in a readonly document or lightweight webview.

  • Compare With Previous Low

    Compare the current file with the previous revision for quick diff review.

  • Branch List Low

    Show local and remote branches in a sidebar view.

  • Commit Search Medium

    Search commits by message, author, short hash, or full hash.

  • Status Panel Medium

    Show changed, staged, and untracked files so users can read the working tree state quickly.

  • Easy Install Low

    Package a .vsix that can be installed from the command line or VS Code UI.

Exit Criteria

  • Compile cleanly with TypeScript, work in normal Git repositories, and avoid GitLens dependencies or third-party branding.

  • Handle files outside Git repositories, missing Git on PATH, renamed files in history, and basic CLI failures gracefully.

Phase 2 Usability

MVP+

This phase improves usability for larger repositories and more complex workspaces.

Priority Features

  • Repo Switcher Medium

    Let users choose the active repository clearly when the workspace contains multiple Git roots.

  • Open File At Revision Medium

    Open a file at a specific commit so users can read historical code without leaving the editor.

  • Line / Selection History Medium

    Trace the history of a single line or selected block instead of only the whole file.

  • Stash / Tags / Remotes Medium

    Add important Git lists so the extension becomes more practical in real repositories.

  • Status Actions Medium

    Allow opening diffs from the status panel and stage, unstage, or discard actions at file level.

  • Peek Diff Preview Medium

    Show diff previews inside the editor without requiring a full diff tab every time.

  • Blame UX Controls Low

    Add quick toggles, relative dates, short hashes, caching, and throttling for large files.

  • Diagnostics & Safeguards Medium

    Improve timeouts, pagination, binary file handling, repo state detection, and user-facing error guidance.

Outcome

  • Users should be able to navigate file history, repository state, and revisions without dropping into the terminal constantly.

Phase 3 Power Features

Advanced

This phase contains more complex features that need heavier UI and stronger guardrails before wide release.

High-Value Features

  • Commit Graph High

    Build a webview graph with node layout, branch decoration, commit selection, and jump-to-details support.

  • Visual File History High

    Create a timeline UI for each file with revision navigation, author context, and diff preview.

  • Worktree Manager Medium

    Manage git worktree operations such as add, list, open, remove, and prune inside VS Code.

  • Interactive Rebase UI High

    Edit rebase plans safely with a todo list UI, conflict state visibility, and rollback guidance.

  • PR Review Panel High

    Integrate GitHub or GitLab APIs for review workflows, comments, checks, and metadata.

  • AI Commit Message Medium

    Generate commit messages from staged diffs using local, Ollama, OpenAI, or custom API providers.

  • AI Commit Explanation Medium

    Explain the meaning of a commit or diff to speed up review and code understanding.

  • Changelog Generator Medium

    Group commits by scope, tag, or conventional commit style to generate changelogs.

  • Conflict Assistant High

    Help users read merge conflicts, explain root causes, and suggest resolution paths.

  • Repository Dashboard Medium

    Provide an overview of branches, remotes, recent commits, ahead/behind state, and repository health.

Rule

  • Only build Advanced features after caching, error handling, multi-repo behavior, and performance are stable in MVP+.

Recommended Build Order

Finish MVP completely first, then build `Repo Switcher`, `Line / Selection History`, and `Status Actions`, before moving into `Commit Graph`, `Worktree Manager`, and `AI` features. This order keeps the extension usable early, reduces complex UI risk, and avoids spending effort on webview-heavy or external-API features before the Git CLI core is stable.
